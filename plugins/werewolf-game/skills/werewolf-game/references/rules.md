# 人狼ゲーム ルールブック

## 勝利条件

| 陣営 | 勝利条件 |
|------|----------|
| 村人陣営 | 人狼を全員処刑する |
| 人狼陣営 | 人狼の生存数 ≥ 村人陣営の生存数 |
| 妖狐陣営 | ゲーム終了時に妖狐が生存している（どちらの陣営が勝っても妖狐が生きていれば妖狐の勝ち） |

## 配役一覧

### 村人陣営

| 役職 | 能力 |
|------|------|
| 村人 | 特殊能力なし。議論と投票で人狼を見つける |
| 占い師 | 毎晩1人を占い、人狼か否かを知る。妖狐を占うと妖狐は死亡（呪殺） |
| 霊能者 | 処刑された者が人狼だったか否かを知る |
| 狩人（騎士） | 毎晩1人を護衛し、人狼の襲撃から守る。自分は護衛できない |

### 人狼陣営

| 役職 | 能力 |
|------|------|
| 人狼 | 毎晩1人を襲撃して殺害する。人狼同士は互いを認識できる |
| 狂人 | 人狼陣営だが占い・霊能結果は「人間」と出る。人狼が誰か知らない |

### 妖狐陣営（9Cのみ）

| 役職 | 能力 |
|------|------|
| 妖狐 | 人狼に襲撃されても死なない。占い師に占われると死亡（呪殺）。処刑されると死亡 |

## 配役構成

### 9A（スタンダード）
- 村人×3、占い師×1、霊能者×1、狩人×1、狂人×1、人狼×2
- 村人陣営6人 vs 人狼陣営3人

### 9B（霊能なし）
- 村人×4、占い師×1、狩人×1、狂人×1、人狼×2
- 村人陣営6人 vs 人狼陣営3人

### 9C（妖狐あり）
- 村人×3、占い師×1、霊能者×1、人狼×2、狂人×1、妖狐×1
- 村人陣営5人 vs 人狼陣営3人 vs 妖狐陣営1人
- ※狩人なしの代わりに妖狐が入る変則構成

## ゲーム進行

### 0日目（夜）
1. GM が各プレイヤーに配役を個別に伝達する
2. 人狼同士に相方を通知する
3. 人狼同士が密談（作戦会議）を行う
4. **占い師の初日お告げ**: 占い師はランダムに1人を占い、結果を得る（白確定）

### 1日目（朝）
1. GM が朝の到来を宣言（「犠牲者はいません」）
2. 議論フェーズ（2周）
3. 投票フェーズ → 最多得票者を処刑
4. 処刑者はGMを通じて遺言を残す（以降は発言しない）

### 1日目（夜）
1. 人狼が襲撃対象を選択
2. 狩人が護衛対象を選択（初日のみ護衛なしでもOK）
3. 占い師が占い対象を選択

### 2日目以降（朝）
1. GM が襲撃結果を発表（護衛成功なら「犠牲者はいません」）
2. 議論フェーズ（2周）
3. 投票フェーズ → 最多得票者を処刑
4. 勝利判定 → ゲーム続行なら夜フェーズへ

## 特殊ルール

- **連続護衛禁止**: 狩人は前の晩と同じ人を護衛できない
- **初日お告げ白あり**: 占い師は0日目夜に1人の白結果を得る
- **同票時**: 決選投票。それでも同票なら引き分け処刑なし（翌日へ）
- **妖狐の呪殺**（9Cのみ）: 占い師が妖狐を占うと翌朝妖狐が死亡する（襲撃死との区別不可）
- **霊能結果の通知**: 夜フェーズ開始時に、当日処刑された者が人狼か人間かを霊能者に通知する（霊能者が死亡している場合はスキップ）
- **襲撃は必須**: 人狼は毎晩必ず誰かを襲撃しなければならない（パスは不可）
- **狂人の勝利判定カウント**: 狂人は人狼陣営だが、勝利判定では「人狼以外の生存者」（人間側）としてカウントする
- **役職の非公開**: 死亡者の役職はゲーム終了まで公開しない。朝の犠牲者発表でも役職は伏せる
- **能力者死亡時**: その能力者のアクションはスキップする（占い師死亡→占いなし、狩人死亡→護衛なし等）

## タイムアウト処理

- **投票の応答なし**: 数回リトライしても応答がない場合、そのプレイヤーの投票は棄権扱い
- **襲撃の応答なし**: 数回リトライしても応答がない場合、GMが `shuf` でランダムに襲撃対象を決定
- **議論の応答なし**: 数回リトライしても応答がない場合、発言をスキップして次の人へ

## 議論ルール

- 議論はターン制で、GM が指定した順番で1人ずつ発言
- 1周 = 全生存者が1回ずつ発言。2周で議論終了
- 発言は他プレイヤーへの質問、推理、弁明、情報共有（COなど）
- CO（カミングアウト）= 自分の役職を公表すること
- 対抗CO = 同じ役職を名乗ること（ローラーの判断材料になる）

### 発言順の決定

プレイヤーには番号順（スポーン順）がある。発言順はその番号順に沿って巡回する（例: 7→8→9→1→2→... のようにラウンドする）。

- **1日目**: ランダムな位置のプレイヤーから開始し、以降は番号順に巡回
- **2日目以降**:
  - 前夜の襲撃で死亡者が出た場合 → 死亡者の**次の番号**の生存者から開始
  - 襲撃が失敗した場合 → 前日の処刑者の**次の番号**の生存者から開始
  - 襲撃も処刑もなかった場合 → 前日と同じ人から開始

## 人狼密談ルール

- 人狼は発言時に **相方とGMの両方** に SendMessage を送る（GMへはログ用）
- 合計 **10メッセージ** が上限（2人の配分は自由。片方が7、もう片方が3でもよい）
- 0日目夜: 作戦会議（10メッセージ）
- 夜の密談: 襲撃対象の相談（10メッセージ）
- 人狼が1人のみ生存している場合: 密談なし、直接襲撃対象を選択
- GM はメッセージカウントを管理し、10に達したら打ち切りを宣言

## 通信ルール

- **すべての通信はGM経由**: プレイヤー同士の直接通信（SendMessage）は禁止。発言・投票・夜のアクションはすべてGMに送信する
- **議論の発言**: プレイヤーがGMに発言を送信 → GMが broadcast で全員に共有
- **投票**: プレイヤーがGMに SendMessage で送信（秘密投票）
- **夜のアクション**: 能力者がGMに SendMessage で送信
- **例外 — 人狼の密談**: 人狼は相方とGMの両方に SendMessage を送る（密談のみ直接通信を許可）

## 死亡プレイヤーの扱い

- 死亡したプレイヤーはゲーム終了まで待機する（shutdown しない）
- 死亡後は一切の発言・投票・能力行使ができない
- ゲーム終了時に全員まとめて退場する

## 処刑時の遺言

- 処刑されるプレイヤーは遺言を1回だけ発言できる
- 遺言はGMに送信し、GMが broadcast で全員に共有する
- 遺言の後、そのプレイヤーは死亡扱いとなり以降は発言しない

## 複数死亡時の朝の宣言

- 襲撃死と呪殺が同時に発生した場合、翌朝に2人の死亡者が出る
- 朝の宣言: 「朝が明けると、○○と○○の死体が発見されました」
- 死因の区別は公開しない（襲撃か呪殺かは伏せる）
